## ❓ TOP 5 Q&A + ANSWERS

**Q1: What exactly is AINL — is it a framework or a language?**
A: It's a language. Not a wrapper around Python, not a prompt template system — an actual compiled language with a grammar, a type system, and a runtime. You write `.ainl` or `.lang` files, the compiler validates them at build time, and the runtime executes deterministic graphs. No LLM reasoning in the orchestration layer. That's where the cost savings come from.

**Q2: How does AINL achieve 90–95% token savings?**
A: Traditional AI agents use an LLM to decide what to do next at every step — route to this tool, handle this error, retry this call. Every one of those decisions burns tokens. AINL eliminates that entirely by encoding the logic in the graph at compile time. The LLM only runs where you explicitly call it for content generation. Everything else is pure graph execution.
